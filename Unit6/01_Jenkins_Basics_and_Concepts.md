# Unit VI — Jenkins: Basics & Concepts

## What is Jenkins?

**Jenkins** is an open-source automation server that helps automate the parts of software development related to:
- **Building** code
- **Testing** code
- **Deploying** applications

It facilitates **Continuous Integration (CI)** and **Continuous Delivery (CD)**.

---

## Why Use Jenkins?

| Feature | Benefit |
|---------|---------|
| Automates builds and tests | No manual triggering needed |
| Integrates with Git, Docker, Maven, Kubernetes | Works with your whole toolchain |
| Supports complex pipelines | End-to-end automation |
| 1800+ plugins | Customizable for any workflow |
| Cross-platform | Works on Linux, Windows, macOS |

---

## Jenkins Key Terms

| Term | Description |
|------|-------------|
| **Job / Project** | A task to be executed (e.g., build a project) |
| **Build** | A single execution/run of a job |
| **Node** | A machine where Jenkins runs jobs (Master or Agent/Slave) |
| **Pipeline** | A series of automated steps defining the CI/CD workflow |
| **Plugin** | Add-on giving Jenkins extra functionality (Git, Docker, Maven plugins) |
