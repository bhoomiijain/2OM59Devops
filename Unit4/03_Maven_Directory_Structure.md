## 📁 Convention Over Configuration

Maven follows a strict directory structure. This convention eliminates configuration needed in pom.xml.

> **Convention Over Configuration:** Maven assumes a standard project layout. If you follow it, minimal configuration needed.

---

## Standard Maven Project Layout

```
my-app/
│
├── pom.xml                          # Project configuration
│
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/
│   │   │       └── example/
│   │   │           ├── Main.java
│   │   │           ├── Service.java
│   │   │           └── Util.java
│   │   │
│   │   ├── resources/
│   │   │   ├── application.properties
│   │   │   ├── application.yml
│   │   │   └── logback.xml
│   │   │
│   │   └── webapp/                  # (WAR projects only)
│   │       ├── WEB-INF/
│   │       ├── index.html
│   │       └── css/, js/, etc.
│   │
│   └── test/
│       ├── java/
│       │   └── com/
│       │       └── example/
│       │           ├── ServiceTest.java
│       │           └── UtilTest.java
│       │
│       └── resources/
│           ├── test-data.sql
│           └── application-test.properties
│
├── target/                          # (Generated by Maven)
│   ├── classes/
│   │   └── (compiled .class files)
│   │
│   ├── test-classes/
│   │   └── (compiled test .class files)
│   │
│   ├── my-app-1.0.0.jar
│   ├── my-app-1.0.0-sources.jar
│   └── (other build outputs)
│
├── README.md
└── .gitignore
```

## Directory Purposes

### src/main/java

**Purpose:** Application source code

```
src/main/java/
├── com/
│   └── example/
│       ├── Main.java            # Entry point
│       ├── controller/
│       │   └── OrderController.java
│       ├── service/
│       │   ├── OrderService.java
│       │   └── PaymentService.java
│       ├── repository/
│       │   └── OrderRepository.java
│       ├── model/
│       │   ├── Order.java
│       │   └── Payment.java
│       └── util/
│           └── DateUtil.java
```

**Compiled to:** `target/classes/`

**Usage:** Included in final JAR/WAR

---

### src/main/resources

**Purpose:** Application configuration and resource files

```
src/main/resources/
├── application.properties        # Main config
├── application.yml              # Alternative YAML format
├── database/
│   └── schema.sql
├── templates/
│   └── email-template.html
├── static/
│   ├── css/
│   ├── js/
│   └── images/
└── logback.xml                  # Logging config
```

**Compiled to:** `target/classes/`

**Usage:** Included in final JAR/WAR (on classpath)

---

### src/test/java

**Purpose:** Unit and integration tests

```
src/test/java/
├── com/
│   └── example/
│       ├── service/
│       │   ├── OrderServiceTest.java
│       │   └── PaymentServiceTest.java
│       ├── repository/
│       │   └── OrderRepositoryTest.java
│       ├── controller/
│       │   └── OrderControllerTest.java
│       └── integration/
│           └── EndToEndTest.java
```

**Compiled to:** `target/test-classes/`

**Usage:** NOT included in final JAR/WAR (test only)

---

### src/test/resources

**Purpose:** Test configuration and test data

```
src/test/resources/
├── application-test.properties  # Test-specific config
├── test-data.sql               # Test database setup
├── fixtures/
│   └── order-data.json         # Test fixtures
└── mock-responses/
    └── api-response.json
```

**Compiled to:** `target/test-classes/`

**Usage:** Available only during test execution

---

### src/main/webapp (WAR Projects Only)

**Purpose:** Web application files (for WAR packaging)

```
src/main/webapp/
├── WEB-INF/
│   ├── web.xml                 # Deployment descriptor
│   ├── lib/                    # Dependencies (auto-included)
│   └── classes/                # Compiled classes (auto-included)
├── index.html
├── js/
│   ├── app.js
│   └── utils.js
├── css/
│   └── style.css
└── images/
    └── logo.png
```

**Not used for JAR projects**

---

### target/ (Generated)

**Purpose:** Build output directory (auto-generated by Maven)

```
target/
├── classes/
│   ├── com/example/Main.class
│   ├── com/example/Service.class
│   └── (compiled bytecode)
│
├── test-classes/
│   ├── com/example/ServiceTest.class
│   └── (compiled test bytecode)
│
├── my-app-1.0.0.jar             # Final artifact
├── my-app-1.0.0-sources.jar     # Source code JAR
├── my-app-1.0.0-javadoc.jar     # JavaDoc JAR
├── surefire-reports/            # Test reports
│   ├── TEST-com.example.ServiceTest.xml
│   └── com.example.ServiceTest.txt
│
├── site/                        # Generated documentation
│   └── index.html
│
└── my-app-1.0.0.pom             # Deployed POM
```

**Always regenerated during build** — safe to delete

```bash
# Safe to delete entire target/ directory
rm -rf target/
# Maven will recreate it on next build
```

## Package Structure Best Practices

### Hierarchical Organization

```
com.example/
├── controller/          # REST endpoints
│   ├── OrderController.java
│   └── PaymentController.java
│
├── service/            # Business logic
│   ├── OrderService.java
│   └── OrderServiceImpl.java
│
├── repository/         # Database access
│   ├── OrderRepository.java
│   └── PaymentRepository.java
│
├── model/              # Data models
│   ├── Order.java
│   ├── Payment.java
│   └── User.java
│
├── exception/          # Custom exceptions
│   ├── OrderNotFoundException.java
│   └── InvalidOrderException.java
│
├── util/               # Utility classes
│   ├── DateUtil.java
│   └── StringUtil.java
│
├── config/             # Configuration
│   ├── DatabaseConfig.java
│   └── SecurityConfig.java
│
└── Main.java          # Entry point
```

**Benefits:**
- ✅ Clear separation of concerns
- ✅ Easy to navigate
- ✅ Follows architectural patterns (MVC, layered, etc.)
- ✅ Standard across Java community

## Naming Conventions

| Item | Convention | Example |
|---|---|---|
| Package | lowercase, dot-separated | `com.example.service` |
| Class | PascalCase | `OrderService` |
| Interface | PascalCase | `OrderRepository` |
| Implementation | ImplSuffix or Prefix | `OrderServiceImpl`, `OrderService` |
| Constant | UPPER_SNAKE_CASE | `MAX_ORDERS = 100` |
| Method | camelCase | `getOrderById()` |
| Field | camelCase | `orderStatus` |
| Test Class | TestSuffix or ItSuffix | `OrderServiceTest`, `OrderServiceIT` |

## Maven vs Gradle Directory Structure

| Aspect | Maven | Gradle |
|---|---|---|
| **Source** | src/main/java | src/main/java |
| **Tests** | src/test/java | src/test/java |
| **Resources** | src/main/resources | src/main/resources |
| **Config File** | pom.xml | build.gradle |
| **Build Output** | target/ | build/ |
| **Flexibility** | Strict | Flexible |

## Creating New Maven Project

### Using Archetype

```bash
# Create new project from template
mvn archetype:generate \
  -DgroupId=com.example \
  -DartifactId=my-app \
  -DarchetypeArtifactId=maven-archetype-quickstart \
  -DarchetypeVersion=1.4 \
  -DinteractiveMode=false

# Result:
# my-app/
#   ├── pom.xml
#   └── src/
#       ├── main/java/com/example/App.java
#       └── test/java/com/example/AppTest.java
```

### Popular Archetypes

| Archetype | Use Case |
|---|---|
| `maven-archetype-quickstart` | Simple JAR application |
| `maven-archetype-webapp` | Web application (WAR) |
| `maven-archetype-j2ee-simple` | J2EE application |

## IDE Integration

### IntelliJ IDEA

```
Recognizes Maven project structure automatically:
✓ src/main/java → Marks as "Sources"
✓ src/test/java → Marks as "Tests"
✓ target/       → Marks as "Excluded"
✓ pom.xml       → Recognizes dependencies
```

### Eclipse

```
Same automatic recognition:
✓ Highlights source/test directories
✓ Shows project dependencies
✓ Marks target/ as excluded
```

### Visual Studio Code

```
Install Maven for Java extension:
✓ Shows Maven project outline
✓ Run Maven commands from UI
✓ Recognizes dependencies
```

## 🔑 Key Takeaway

> **Maven's standard directory structure eliminates configuration. Follow the convention, and Maven handles the rest.**

**Benefits:**
1. ✅ No configuration needed (works out-of-box)
2. ✅ Consistent across all Maven projects
3. ✅ IDE automatically recognizes structure
4. ✅ Easy for new developers to navigate
5. ✅ Standard Java community practice

**Remember:**
```
src/main/java        → Your code
src/main/resources   → Config files
src/test/java        → Tests (NOT in JAR)
src/test/resources   → Test config
target/              → Build outputs (generated)
```
